2 body
14.

Uvažte následující Vue.js aplikaci:

<div id="app">
  <button
    class="btn"
    v-on:click="btnClick"
  >
  Ok
  </button>
</div>
<script src="vue.js"></script>
<script>
  new Vue({
    el: '#app',
    data: {
      active: true,
    },
    methods: {
      btnClick() {
        this.active = !this.active;
      }
    }
  });
</script>

Aplikace při každém kliknutí na tlačítko mění hodnotu vlastnosti active z true na false a obráceně.

Chceme, aby se tato změna projevila vizuálně i na tlačítku. Předpokládejme, že v CSS máme nastylovanou třídu btn-active. Když tuto třídu přidáme na tlačítko, bude tlačítko zvýrazněné jako aktivní.

Upravte aplikaci uvedenou výše tak, aby se na tlačítku objevila CSS třída btn-active, je-li hodnota vlastnosti active nastavena na true. Je-li hodnota vlastnosti active nastavena na false, třída na tlačítku nebude.

Jako řešení stačí poslat pouze řádky se změněným kódem. Mělo by se jednat pouze o 1-3 řádky.

16.

Vezměte projekt z předchozí otázky a postupujte podle podrobného zadání v souboru README.md, který je součástí projektu. Zadání si můžete prohlédnout také online na adrese:
xxxxx.yyy

Stručně:

  1. V App.vue vytvořte v datech seznam vtipů - pole objektů, kde každý objekt představuje jeden vtip, který má následující vlastnosti:
    • text vtipu
    • název ikonky osoby, která se má u vtipu zobrazit
    • počet lajků (kolikrát se vtip líbil)
    • počet nelajků (kolikrát se vtip nelíbil)
    Seznam vtipů najdete v kořenové složce projektu v souboru vtipy.txt nebo na adrese xxxxx.yyy.
  2. Vytvořte komponentu Joke, která bude zobrazovat jeden vtip. Vtip, který se má v komponentě zobrazit, dostane komponenta jako props. V komentáři uvnitř App.vue najdete zakomentovaný kód pro HTML strukturu jednoho vtipu. Z toho HTML vytvořte šablonu komponenty Joke.
  3. V HTML šabloně uvnitř App.vue zobrazte pod sebou všechny vtipy - tj. opakujte komponentu Joke pro každý vtip ze seznamu.
  4. Uvnitř komponenty Joke zařiďte, aby se klikáním na palec nahoru nebo dolů zvětšoval počet příslušných lajků/nelajků.
  5. Komponenta by neměla měnit svoje vlastní data (získana z props), ale požadavek na změnu počtu lajků emitovat do mateřské komponenty App.vue, kde teprve ke změně dat dojde.

Výsledná aplikace by měla zobrazovat seznam všech vtipů se správnými ikonami uživatelů a počty lajků. Tlačítka u každého vtipu by měla správně fungovat.
